Week 5 Technical Blog: Classes in Ruby

As you may or may not already know, everything in Ruby is an object! It's true! Look it up! Or could just trust me. I've done the research.

Each object in ruby has it's own properties and methods. Many of the built-in methods allow us to do some pretty powerful stuff. But what if we want to add our own methods? Or better yet, what if we want to create our own objects? We can do that! Check it out:

class Dog
  def initialize(name, sex, breed, weight, fur_color)
    @name = name
    @sex = sex
    @breed = breed
    @weight = weight #in pounds
    @fur_color = fur_color
    @pronoun = @sex == "male" ? "his" : "her"
  end

  def dig
    "#{@name} dug a hole, and #{@pronoun}"\
    " #{@fur_color} fur is now the color"\
    " of dirt!"
  end
end

Cool! We began by creating a new class: Dog. Inside this class, we definied an initialize method, which takes a number of parameters for when a new Dog is created. All the @ signs? Those are instance variables. Each instance of Dog will have those variables defined. We also created a dig method, which accesses a number of instance variables and returns a string. Let's create a new Dog and see what happens

max = Dog.new("Max", "male", "Westie", 20, "white")
puts max.dig

==> Max dug a hole, and his white fur is now the color of dirt!

That worked really well! But what if we wanted to create another dog who didn't dig, but instead liked to catch frisbees? We could add a catch_frisbee method to the original Dog class, but it's doubtful that our original dog, max, would ever need to do that. This is where inheritance will be useful.

Each class or object in Ruby inherits certain properties and methods from it's parent. Even without specifying it, our Dog class has a parent, and has a number of methods already built in which we didn't define. However, now that we have created our Dog class, we can also create other classes which inherit methods and properties from it. Let's refactor our code a bit to accomodate a new, frisbee-catching dog.

class Dog
  def initialize(name, sex, breed, weight, fur_color)
    @name = name
    @sex = sex
    @breed = breed
    @weight = weight #in pounds
    @fur_color = fur_color
    @pronoun = @sex == "male" ? "his" : "her"
  end
end

class Terrier < Dog
  def dig
    "#{@name} dug a hole, and #{@pronoun}"\
    " #{@fur_color} fur is now the color"\
    " of dirt!"
  end
end

class Collie < Dog
  def catch_frisbee
    "#{@name} chases after a frisbee at full"\
    " sprint, leaps in the air, and catches it"\
    " in #{@pronoun} mouth!"
  end
end

Now we have two specific types of classes, Terrier and Collie, which both inherit the initialize method and instance variables from Dog. The Terrier class has it's own dig method, and the Collie class has it's own catch_frisbee method. Let's create a couple of examples and see how they work!

max = Terrier.new("Max", "male", "Westie", 20, "white")
puts max.dig

==> Max dug a hole, and his white fur is now the color of dirt!

pepper = Collie.new("Pepper", "female", "Border Collie", 35, "black")
puts pepper.catch_frisbee

==> Pepper chases after a frisbee at full sprint, 
    leaps in the air, and catches it in her mouth!

They work! If Max tried to catch a frisbee, or Pepper tried to dig, they would run into an error. While this may not be a perfect example, it certainly shows how classes can inherit certain properties from their parent classes, and then branch out on their own to accomplish specific tasks.
