Week 4 Technical Blog: Enumerable#cycle

For this week's techincal blog, I will be highlighting the uses of Enumberable#cycle in Ruby. I was interested in the method because I had never used or heard of it before! Luckily, it is fairly simple to use and understand.

Enumerable#cycle can be called on any enumerable object (such as a range, hash, or array) just like other familiar enuerable methods (like #each or #map). When called, a block will be run, either n times or forever (if no number is passed as an argument).

a = [1, 2, 3]
b = {1: a, 2: b, 3: c}
a.cycle { |x| puts x } # ==> print 1, 2, 3, 1, 2, 3... forever.
b.cycle(2) { |k, v| puts v } # ==> print a, b, c, a, b, c.

Obviously, more complex code can be called within the block than what was just shown, but it serves as a good illustration. Be extremely careful calling #cycle without passing a number as an argument, as it will result in an infinite loop if you do not designate a break or stopping condition inside the loop (as I learned the hard way!)
