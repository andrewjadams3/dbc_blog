Inline vs. Inline Block
May 4, 2014
This week, I will explore the differences between inline elements and inline-block elements. To gain a better understanding, let's first take a look at plain-old block elements.

Block
Block elements are characterized by a line break before and after the element. This means that block elements cannot sit side-by-side. For example, if you created two paragraphs (which are default block-elements) like so: 
<p>Paragraph 1</p>
<p>Paragraph 2</p>
they would sit on top of each other, not next to each other. Block elements can have set height, width, margin, and padding, which are all respected by surrounding elements.

Inline
Inline elements are essentially the opposite of block elements. These elements do not force line breaks, meaning that other elements can sit next to them. Using the paragraph example from before, we can add the following lines of CSS to force the paragraphs to sit next to each other: 
p {
  display: inline;
}
We can even add some margin to space them apart: 
p {
  display: inline;
  margin: 10px;
}
However, it is important to note that while left and right margins are respected by other elements, the top and bottom margins are not. Inline elements also cannot have a set height or width. That's where inline-block comes in.

Inline-Block
Inline-block elements contain the best parts of block elements and inline elements. By setting our paragraphs to inline-block... 
p {
  display: inline-block;
  margin: 10px;
}
...we allow them to not only sit next to each other, but also force any elements above or below to respect the 10px margin. This means that inline-block elements behave very similarly to floats. However, unlike floats, inline-block elements will not cause any issues with collapsing parent containers. Pretty cool!
